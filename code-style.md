# Code Style Guidelines

Code must prioritize maintainability and simplicity.

---

# Naming Conventions

Use clear descriptive names.

Examples:

getProducts  
updateInventory  
createOrder  

Avoid unclear abbreviations.

---

# File Structure

Projects should follow a clear structure:

/components  
/pages  
/api  
/styles  
/admin  

---

# Comments

Important logic must include comments explaining:

- purpose
- behavior
- assumptions

---

# Readability

Code must prioritize readability over cleverness.

Future developers must easily understand the code.

---

# Performance

Avoid unnecessary dependencies.

Keep the code lightweight to ensure fast loading for users with slower internet connections.
